\subsection{Обзор существующих программных продуктов по теме работы}

На сегодняшний день на рынке представлен ряд программных продуктов для
автоматизации управления фитнес-центрами. Для проведения сравнительного
анализа были выделены следующие критерии оценки:

\begin{itemize}
	\item наличие онлайн-записи на тренировки;
	\item разграничение прав доступа по ролям пользователей;
	\item наличие личного кабинета клиента;
	\item возможность загрузки файлов (аватар, фото тренировок);
	\item стоимость и доступность решения для небольших студий.
\end{itemize}

табл.~\ref{tab:competitors}.

Проведённый анализ позволил выявить следующие недостатки существующих решений:

\begin{itemize}
	\item \textbf{Mindbody}~\cite{mindbody} --- ориентирован на крупные сети,
	      высокая стоимость подписки, избыточный функционал для небольших студий,
	      отсутствует загрузка файлов пользователями.
	\item \textbf{1С:Фитнес клуб} --- требует установки и настройки на сервере,
	      отсутствует онлайн-запись и личный кабинет клиента, высокая стоимость
	      лицензии.
	\item \textbf{Fitness Pro} --- нет разграничения ролей пользователей,
	      отсутствует загрузка файлов, ограниченная кастомизация.
\end{itemize}

Таким образом, ни одно из рассмотренных решений не обеспечивает в полной мере
потребности небольшого фитнес-центра. Разрабатываемое приложение компенсирует
выявленные недостатки: предоставляет онлайн-запись, разграничение прав по ролям,
личный кабинет с историей записей и загрузкой файлов при свободном доступе
к исходному коду.

\subsection{Формулировка цели и задач работы}

Анализ существующих программных продуктов показал, что доступные решения
для автоматизации фитнес-центров либо ориентированы на крупные сети и имеют
высокую стоимость, либо не обеспечивают полного набора необходимых функций:
онлайн-записи, разграничения ролей и загрузки файлов. Анализ инструментов
разработки позволил обосновать выбор технологического стека.

\textbf{Цель работы} --- проектирование и реализация веб-приложения для
фитнес-центра, обеспечивающего онлайн-запись клиентов на тренировки,
управление расписанием и личный кабинет пользователя с разграничением
прав доступа по ролям.

Для достижения поставленной цели необходимо решить следующие \textbf{задачи}:
\begin{enumerate}
	\item Провести анализ существующих программных продуктов и обосновать
	      выбор технологического стека;
	\item Спроектировать модель данных и архитектуру приложения;
	\item Разработать систему аутентификации, авторизации и управления ролями
	      пользователей (клиент, тренер, администратор);
	\item Реализовать CRUD-интерфейс для управления тренировками, расписанием
	      и пользователями;
	\item Реализовать онлайн-запись на тренировки с проверкой доступных мест
	      и запретом двойной записи;
	\item Разработать личный кабинет пользователя с историей записей и
	      возможностью загрузки файлов;
	\item Реализовать агрегацию данных для формирования статистики посещаемости.
\end{enumerate}
